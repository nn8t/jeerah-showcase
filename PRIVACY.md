# Jeerah Privacy

> Public privacy overview for **Jeerah**, a smart trip-pooling delivery platform for remote communities.

---

## Repository Notice

This file provides a high-level privacy overview for the Jeerah public showcase repository.

It is not a final production privacy policy, legal document, or regulatory compliance statement. A complete production privacy policy should be reviewed before public launch.

---

## Privacy Philosophy

Jeerah is designed around the principle of collecting and showing only the data required to complete delivery operations safely and efficiently.

The platform should protect customer, driver, order, payment, and operational information.

---

## Data Categories

Jeerah may process the following categories of data in a production environment:

| Category | Examples |
|---|---|
| Account Data | Phone number, user identity, role |
| Customer Data | Delivery requests, order history, delivery destination |
| Driver Data | Driver profile, assigned trips, delivery actions |
| Order Data | Order details, status, lifecycle events |
| Trip Data | Shared trip state, linked orders, driver actions |
| Invoice Data | Merchant invoice amount, optional invoice image |
| Payment State Data | Payment method selection and payment status |
| Operational Data | Admin review, support context, workflow events |

---

## Public Repository Data

This public repository does not include:

- Real customer data
- Real driver data
- Real order records
- Real addresses
- Real phone numbers
- Real payment data
- Real invoice images
- Production logs
- Production database exports
- Production screenshots with sensitive information

---

## Privacy Principles

Jeerah should follow these privacy principles:

- Collect only necessary data.
- Limit access based on role.
- Avoid exposing sensitive information in public materials.
- Use demo data for screenshots and documentation.
- Protect invoice and payment-related data.
- Keep production logs private.
- Avoid storing secrets in user-visible files.
- Provide clear privacy terms before public launch.

---

## Role-Based Data Visibility

| Role | Public Privacy Principle |
|---|---|
| Customer | Should only access their own orders and relevant delivery updates |
| Driver | Should only access information needed to complete assigned trips |
| Admin | Should access operational information based on permission level |
| Backend | Should enforce access and validation rules |

---

## Screenshot Privacy Rules

Any public screenshots should be sanitized.

Do not show:

- Real names
- Phone numbers
- Addresses
- Payment details
- Invoice details
- Internal IDs
- Driver personal data
- Admin-only secrets
- Production URLs
- API responses
- Logs

---

## Payment Privacy

Payment-related information should be treated as sensitive.

This repository does not include:

- Payment provider configuration
- Payment verification logic
- Payment tokens
- Webhook secrets
- Transaction details
- Customer financial data
- Reconciliation details

---

## Future Privacy Work

---

## Summary

This public showcase repository does not contain private user data.

A complete production privacy policy should be prepared before Jeerah launches publicly.
