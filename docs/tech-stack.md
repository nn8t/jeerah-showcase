# Jeerah Tech Stack

> Public technology stack overview for Jeerah.

---

## Overview

Jeerah is built using a modern mobile-first stack suitable for commercial MVP development and future scalability.

---

## Stack Summary

| Layer | Technology |
|---|---|
| Mobile | Flutter |
| Language | Dart |
| Backend Platform | Supabase |
| Database | PostgreSQL |
| Server Functions | Edge Functions |
| Backend Language | TypeScript |
| Runtime | Deno |
| Authentication | Phone OTP |
| Design | Figma |
| Version Control | Git / GitHub |

---

## Mobile

### Flutter

Flutter is used for the customer and driver mobile applications.

Reasons:

- Cross-platform development
- Fast iteration
- Strong UI capabilities
- Single codebase for multiple platforms
- Good fit for startup MVPs

### Dart

Dart is used as the primary language for Flutter development.

---

## Backend

### Supabase

Supabase is used as the backend foundation.

Publicly described capabilities:

- Authentication
- PostgreSQL database
- Storage
- Realtime updates
- Edge Functions

### Edge Functions

Edge Functions support server-side workflows and validation.

Private function code is not published.

---

## Database

### PostgreSQL

PostgreSQL is used because Jeerah depends on structured relationships between:

- Users
- Drivers
- Orders
- Trips
- Invoices
- Payments
- Delivery events
- Admin operations

---

## Authentication

Phone OTP authentication is used for mobile-first onboarding.

---

## Design Tools

Figma is used for UI/UX design and prototyping.

---

## Version Control

Git and GitHub are used for source control and public showcase documentation.

Production code remains private.

---

## Not Publicly Included

This repository does not include:

- Flutter source code
- Backend source code
- Database schema
- Supabase configuration
- API keys
- Payment code
- Private business logic
