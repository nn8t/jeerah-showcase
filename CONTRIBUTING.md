# Contributing to Jeerah

> Contribution guidelines for **Jeerah**, a closed-source commercial showcase repository.

---

## Repository Notice

Jeerah is a private commercial product under active development.

This public repository is a showcase repository only. It is intended to present the project's product vision, high-level architecture, system design, roadmap, security approach, and development progress.

The production source code is not included and external code contributions are not accepted at this time.

---

## Current Contribution Policy

Jeerah is currently:

- Closed source
- Commercially owned
- Under active private development
- Not accepting external source code contributions
- Not accepting pull requests for implementation changes

This repository may accept limited feedback related to public documentation only.

---

## What You Can Contribute

You may provide feedback on:

- Documentation clarity
- Typographical errors
- Broken internal links
- Suggestions for improving public explanation
- High-level architecture questions
- Portfolio presentation suggestions

---

## What You Cannot Contribute

Please do not submit:

- Source code
- Backend implementation
- Database schema
- SQL migrations
- Supabase configuration
- Edge Functions
- Pricing logic
- Trip-pooling logic
- Payment implementation
- Driver earnings formulas
- Admin permission logic
- Deployment scripts
- Environment variables
- API keys
- Secrets
- Reverse-engineered implementation

---

## Pull Request Policy

Pull requests are not accepted for production implementation.

If pull requests are enabled later, they will be limited to documentation-only updates.

Any submitted PR that attempts to add source code, expose private implementation, or modify the commercial structure of the project may be closed without review.

---

## Issue Policy

Issues may be used only for public documentation feedback.

Suitable issue examples:

- A typo in README.md
- A broken link
- A confusing public diagram
- A documentation consistency issue

Unsuitable issue examples:

- Requests for source code
- Requests for database schema
- Requests for deployment instructions
- Requests for payment logic
- Requests for trip-pooling algorithm
- Requests for private business rules

---

## Security Reports

Do not open public issues for security concerns.

If you believe sensitive information was accidentally exposed in this showcase repository, report it privately to the project owner.

---

## Commercial Protection

All Jeerah materials remain proprietary.

No contribution, suggestion, issue, or discussion grants any rights to use, copy, clone, reproduce, redistribute, or commercialize Jeerah.

---

## Summary

Jeerah is not currently accepting external contributions.

This repository is maintained as a professional showcase for a private commercial software product.
